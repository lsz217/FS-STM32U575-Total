## 一、架构概述

SensorPage 包含 4 个传感器图标，每个图标下方有一个不可见的点击区域。点击图标弹出对应的 ModalWindow 显示实时传感器数据。

```
屏幕布局 (320×240):

  [湿度图标]      [CO2图标]      [温度图标]
  (0,0) 55×55    (145,0) 56×55  (259,5) 59×50
  
                    [心率图标]
                  (146,120) 55×55
                                        [返回]
                                      (261,186)
```

### 数据流

```
硬件传感器 → Model (tick轮询) → Presenter → View (updateSensorInfo)
                                                      ↓
                                        写入 wildcard buffer → textArea.invalidate()
```

### 点击流

```
用户点击图标区域
    → flexButton (透明覆盖层, 50×50)
    → flexButtonCallbackHandler
    → showTemp() / showHum() / showCO2() / showHeartRate()
    → hideActiveModal() + modalWindow.setVisible(true) + invalidate()
```

## 二、SVG 图标导致界面卡死

### 问题现象

点击图标下方的 flexButton 后，按钮显示按下状态，但 ModalWindow 无法弹出，整个界面卡死。

### 根本原因

**SVG 矢量渲染 + ModalWindow 半透明 shade 叠加，超出 CanvasWidgetRenderer 处理能力。**

分析链路：

1. SensorPage 有 4 个 `SVGImage`，每个都需要 `CanvasWidgetRenderer` 软件渲染
2. CanvasWidgetRenderer 的缓冲区仅 4800 字节，位于 `canvasBuffer[CANVAS_BUFFER_SIZE]`
3. 4 个 SVG 同时可见，渲染面积合计约 11,000 像素（54×50 + 53×50 + 60×50 + 62×55）
4. 当 ModalWindow 的 shade（`setShadeAlpha(150)` 半透明黑色遮罩）弹出时，TouchGFX 渲染管线需要：
   - 重新渲染底层所有 SVG 内容（通过 CanvasWidgetRenderer）
   - 将 SVG 渲染结果与 shade 颜色做帧缓冲混合
   - 叠加 ModalWindow 的背景位图
5. STM32U575 是 Cortex-M33 无硬件 GPU，全部软件渲染。SVG → 帧缓冲 → shade 混合 这一管线在 4800 字节缓冲下无法完成，渲染管线阻塞导致界面冻结

### 解决方案

将 4 个 `SVGImage` 替换为 `ScalableImage`（PNG 位图），彻底移除 `CanvasWidgetRenderer` 依赖。

**对比：**

| | 旧方案 (SVGImage) | 新方案 (ScalableImage) |
|---|---|---|
| 渲染方式 | CanvasWidgetRenderer 软件矢量渲染 | 直接帧缓冲 blit |
| 缓冲区需求 | 需 canvas buffer（至少数KB） | 无额外缓冲 |
| 与 shade 混合 | 需要额外合成步骤 | 正常 z-order 绘制 |
| CPU 开销 | 高（路径计算 + 填充 + 混合） | 低（内存拷贝） |
| 修改方式 | — | TouchGFX Designer 替换控件 |

### 修改步骤

1. 在 TouchGFX Designer 中删除 4 个 SVGImage
2. 添加 4 个 ScalableImage，位置尺寸不变，关联对应的 PNG 位图
3. 删除不再使用的 SVG 文件（`assets/images/*.svg`）
4. Generate Code + Generate Assets

## 三、图片中文文件名导致 Generate Assets 失败

### 问题现象

```
ERROR: Filename must consist of alpha numeric characters (A-Z, a-z, 0-9) 
       and underscore (_): 'assets/images/mti-温度 (1).png'
make: *** [assets] Error 2
```

### 根本原因

TouchGFX 的 Imageconverter 工具（`make assets` 流程）只接受 ASCII 文件名。中文、空格、连字符在部分位置都会触发校验失败。

### 解决方案

重命名 5 个中文文件名：

| 原文件名 | 新文件名 | 用途 |
|---|---|---|
| `mti-温度 (1).png` | `mti_temp_1.png` | 温度图标 |
| `湿度 (1).png` | `humidity_1.png` | 湿度图标 |
| `二氧化碳 (1).png` | `co2_1.png` | CO2 图标 |
| `心率 (2).png` | `heart_rate_2.png` | 心率图标 |
| `心率 (1).png` | `heart_rate_1.png` | 心率（备用） |

同时更新 `.touchgfx` 项目文件中的 `"Filename"` 字段为新文件名。Generate Assets 后 BitmapDatabase 会生成正确的 ID：

| 文件 | 生成的 Bitmap ID |
|---|---|
| `humidity_1.png` | `BITMAP_HUMIDITY_1_ID` |
| `co2_1.png` | `BITMAP_CO2_1_ID` |
| `mti_temp_1.png` | `BITMAP_MTI_TEMP_1_ID` |
| `heart_rate_2.png` | `BITMAP_HEART_RATE_2_ID` |

## 六、FlexButton 点击无响应 — alpha 穿透问题

### 问题现象

flexButton 点击后没有任何反应，`flexButtonCallbackHandler` 不被调用。

### 排查过程

1. **回调签名正确性** — `Callback<SensorPageViewBase, const AbstractButtonContainer&>` 与 `BoxWithBorderButtonStyle<ClickButtonTrigger>` 的 `setAction` 签名匹配。对照 `startuppageViewBase` 相同模式正常工作，排除。

2. **Z-order 正确性** — flexButton 在 `add()` 顺序中位于 ScalableImage 之后（上层），应优先接收触摸，排除。

3. **ModalWindow 遮挡** — 初始状态为 `hide()`（`setVisible(false)`），不可见控件不参与触摸分发，排除。

4. **ScalableImage 拦截** — `ScalableImage` 继承自 `Widget`，`isTouchable()` 默认返回 `false`，不消耗触摸事件，排除。

5. **`setAlpha()` 的影响** — 这是关键发现。

### 根本原因

**TouchGFX 的触摸分发机制中，`Drawable::isSolid()` 检查 `alpha == 255`。当 widget 的 alpha 不等于 255 时，该 widget 被视为"非实体"，点击事件会穿透到下层控件。**

原来的代码：
```cpp
flexButton1.setAlpha(100);  // 100 < 255 → 非实体 → 点击穿透！
```

即使 alpha=100（半透明可见），点击也会被 TouchGFX 框架跳过，传递给下层的 ScalableImage（不处理点击）或更下层的背景 Image，导致回调永远不被触发。

### 解决方案

```cpp
flexButton1.setAlpha(255);  // 必须为 255 才能接收点击
```

## 七、透明点击区的实现

### 问题

`setAlpha(255)` 使 flexButton 能够接收点击，但 `BoxWithBorderButtonStyle` 会绘制可见的蓝色边框/填充，遮挡下方的图标。

### 解决方案

将 `BoxWithBorderButtonStyle` 的绘制盒尺寸设为 0×0，使其不产生任何视觉输出，同时 widget 的 position 保持 50×50 作为点击热区。

```cpp
flexButton1.setBoxWithBorderPosition(0, 0, 0, 0);  // 绘制盒尺寸为 0 → 不可见
flexButton1.setAlpha(255);                           // alpha=255 → 可点击
flexButton1.setPosition(5, 0, 50, 50);               // 点击热区
flexButton1.setAction(flexButtonCallback);
add(flexButton1);
```

不需要设置 `setBorderSize()` 和 `setBoxWithBorderColors()`，因为绘制盒为 0×0 什么都不画。

**原理示意：**

```
┌──────────────────────────┐
│ ScalableImage (图标)     │  ← 可见，不可点击
│   ┌────────────────┐     │
│   │ flexButton     │     │  ← 不可见，50×50 点击热区
│   │ 绘制盒 0×0    │     │     alpha=255 保证 TouchGFX 分发点击事件
│   │ alpha=255      │     │     绘制盒 0×0 保证无视觉输出
│   └────────────────┘     │
└──────────────────────────┘
```

## 八、ModalWindow 显示/隐藏逻辑

### 设计原则（参照 startuppage）

- 使用 `setVisible(true/false)` 替代 `show()/hide()`
- 每次状态变更后显式调用 `invalidate()` 确保重绘
- 只操作实际需要变化的控件，避免遍历所有 4 个 ModalWindow

### SensorPageView.hpp

```cpp
class SensorPageView : public SensorPageViewBase
{
public:
    SensorPageView();
    virtual ~SensorPageView() {}
    virtual void setupScreen();
    virtual void tearDownScreen();
    virtual void backToHome();
    virtual void showTemp();
    virtual void showHum();
    virtual void showCO2();
    virtual void showHeartRate();
    virtual void updateSensorInfo(float temperature, float humidity,
                                   uint16_t co2, uint32_t heartRate, uint32_t spo2);
private:
    void hideActiveModal();     // 只隐藏当前活跃的 modal
    void hideAllModals();       // 隐藏所有 + invalidate
    int8_t activeModal;         // -1=无, 0=温度, 1=湿度, 2=CO2, 3=心率
};
```

### SensorPageView.cpp 核心逻辑

```cpp
void SensorPageView::hideActiveModal()
{
    // 只隐藏当前活跃的 modal，不遍历全部
    if      (activeModal == 0) modalWindow1.setVisible(false);
    else if (activeModal == 1) modalWindow2.setVisible(false);
    else if (activeModal == 2) modalWindow3.setVisible(false);
    else if (activeModal == 3) modalWindow4.setVisible(false);
}

void SensorPageView::hideAllModals()
{
    hideActiveModal();
    activeModal = -1;
    invalidate();           // 统一重绘，不依赖内部隐式 invalidate
}

void SensorPageView::showTemp()
{
    if (activeModal == 0)
    {
        // 点击已打开的图标 → 关闭（toggle）
        modalWindow1.setVisible(false);
        activeModal = -1;
    }
    else
    {
        // 关闭旧的，打开新的
        if (activeModal >= 0) hideActiveModal();
        modalWindow1.setVisible(true);
        activeModal = 0;
    }
    invalidate();           // 模仿 startuppage function5() 的显式 invalidate
}
// showHum / showCO2 / showHeartRate 同理
```

### 与旧版代码的对比

| | 旧版 | 新版 |
|---|---|---|
| 隐藏方式 | `hideAllModals()` 遍历 4 个 | `hideActiveModal()` 只隐藏 1 个 |
| API | `show()` / `hide()` | `setVisible()` + `invalidate()` |
| invalidate | 依赖 show/hide 内部调用 | 显式调用，行为可控 |
| 点击已打开图标 | 关闭再打开（时序冗余） | 直接关闭（toggle） |
