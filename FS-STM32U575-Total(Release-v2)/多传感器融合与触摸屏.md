## 问题一：SensorPage 屏幕亮度太低，看不到文字

**现象**：进入 SensorPage 后，整体亮度太低，看不清温度、湿度等文字。

**原因**：背光 PWM 默认占空比为 50%，即 `gBacklightVal = 50`，对应 PWM 比较值 500（ARR=999）。

**解决**：将背光默认值从 50 改为 90。

**修改文件**：

- `Core/Src/user_app.c` 第 41 行：
```c
volatile uint8_t gBacklightVal = 90;  // 原值 50 → 90
```

- `Core/Src/main.c` 第 241-242 行：
```c
gBacklightVal = 90;
Update_Backlight(90);
```

**背光驱动说明**（`user_app.c` 第 638-650 行）：
```c
void Update_Backlight(uint8_t pDutyRatio)
{
    if((pDutyRatio < 5) || (pDutyRatio > 100)) return;
    HAL_TIM_PWM_Stop(&htim2, TIM_CHANNEL_3);
    __HAL_TIM_SET_COMPARE(&htim2, TIM_CHANNEL_3, pDutyRatio * 10);  // 5-100 → 50-1000
    __HAL_TIM_SET_COUNTER(&htim2, 0);
    HAL_TIM_PWM_Start(&htim2, TIM_CHANNEL_3);
}
```
TIM2 CH3 → PWM 背光，PSC=63, ARR=999，占空比 0%-100%。

## 问题三：编译错误 — `BITMAP_IMAGE_MENU_ICON_ID` 未定义

**现象**：
```
error: 'BITMAP_IMAGE_MENU_ICON_ID' was not declared in this scope
error: 'BITMAP_IMAGE_MENU_ICON_PRESSED_ID' was not declared in this scope
```

**原因**：`.touchgfx` 文件中 ButtonBack 引用了 `Controls/image_menu_icon.png`，TouchGFX Designer 会根据文件名自动生成对应的 Bitmap ID `BITMAP_IMAGE_MENU_ICON_ID`，但实际存在的 bitmaps 文件名是 `Controls/menu_icon.png`，生成的是 `BITMAP_MENU_ICON_ID`。Designer 每次 "Generate Code" 会覆盖手动修改。

**解决**：创建物理文件副本，使两个命名同时存在：
```
Controls/menu_icon.png          →  BITMAP_MENU_ICON_ID
Controls/image_menu_icon.png    →  BITMAP_IMAGE_MENU_ICON_ID  (menu_icon.png 的副本)
Controls/menu_icon_pressed.png       →  BITMAP_MENU_ICON_PRESSED_ID
Controls/image_menu_icon_pressed.png →  BITMAP_IMAGE_MENU_ICON_PRESSED_ID (副本)
```

同时手动修改生成的 `SensorPageViewBase.cpp` 中的 bitmap 引用确保编译通过。

## 问题四：SVG 文件代码生成失败

### 4.1 文件名验证错误

**现象**：
```
Error: Filename must consist of alpha numeric characters (A-Z, a-z, 0-9) and underscore (_)
```

**原因**：SVG 文件名包含中文字符和空格：`mti-温度.svg`、`二氧化碳 (1).svg`、`湿度.svg`、`24gf-heartPulse (1).svg`

**解决**：重命名所有 SVG 文件为合法名称：
```
mti-温度.svg          → temp_icon.svg
二氧化碳 (1).svg      → co2_icon.svg
湿度.svg              → humidity_icon.svg
24gf-heartPulse (1).svg → heartPulse_24gf.svg
```

删除不再使用的文件：`24gf-heartPulse.svg`、`二氧化碳.svg`、`空气湿度.svg`

### 4.2 SVG 属性解析错误

**现象**：
```
Error: Wrong svg attribute 't'
Error: Wrong svg attribute 'p-id'
Error: Wrong svg attribute 'class'
```

**原因**：SVG 文件中包含 TouchGFX Designer 不支持的 XML 属性。

**解决**：使用 sed 批量移除非法属性：
```bash
sed -i 's/ t="[^"]*"//g' temp_icon.svg co2_icon.svg humidity_icon.svg heartPulse_24gf.svg
sed -i 's/ p-id="[^"]*"//g' temp_icon.svg co2_icon.svg humidity_icon.svg heartPulse_24gf.svg
sed -i 's/ class="[^"]*"//g' temp_icon.svg co2_icon.svg humidity_icon.svg heartPulse_24gf.svg
```

## 问题五：链接错误 — `VectorRenderer::getInstance()` 未定义

**现象**：
```
Error: L6218E: Undefined symbol touchgfx::VectorRenderer::getInstance() 
        (referred from svgimage.o)
```

**原因**：`SVGImage` 类内部调用 `VectorRenderer::getInstance()` 来获取矢量渲染器实例。TouchGFX Simulator 在 `mainBase.cpp` 中提供了该实现，但目标硬件工程中缺失。预编译库 `touchgfx_core_wchar16.lib` 不包含 SVG/Vector 渲染器相关代码。

**解决**：在 `TouchGFXConfiguration.cpp` 中添加实现：

**修改文件** `TouchGFX/target/generated/TouchGFXConfiguration.cpp`：

包含头文件：
```cpp
#include <touchgfx/widgets/canvas/CWRVectorRenderer.hpp>
#include <touchgfx/widgets/canvas/PainterRGB565.hpp>
```

在文件末尾添加（before `END OF FILE`）：
```cpp
namespace touchgfx
{
    VectorRenderer* VectorRenderer::getInstance()
    {
        static CWRVectorRendererRGB565 renderer;
        return &renderer;
    }
}
```

> `CWRVectorRendererRGB565` 对应 16-bit RGB565 帧缓冲格式（ILI9341 LCD）。

## 问题六：Keil 工程缺少 SVG/Canvas 框架源文件

**现象**：虽然 `VectorRenderer::getInstance()` 已实现，但 `SVGImage` 和 `CWRVectorRenderer` 的类方法仍然未在工程中编译，导致链接错误。

**原因**：Keil uVision 工程 `FS-STM32U575-Total.uvprojx` 中只链接了预编译库 `touchgfx_core_wchar16.lib`，而该库不包含 SVG/Canvas 框架组件。

**解决**：在 Keil 工程 XML 中新增 `TouchGFX/Framework` 组，添加所需源文件：

**修改文件** `MDK-ARM/FS-STM32U575-Total.uvprojx`：

在 `<Group><GroupName>Lib</GroupName>` 之前插入：
```xml
<Group>
  <GroupName>TouchGFX/Framework</GroupName>
  <Files>
    <File>
      <FileName>SVGImage.cpp</FileName>
      <FileType>8</FileType>
      <FilePath>../Middlewares/ST/touchgfx/framework/source/touchgfx/widgets/SVGImage.cpp</FilePath>
    </File>
    <File>
      <FileName>CWRVectorRenderer.cpp</FileName>
      <FileType>8</FileType>
      <FilePath>../Middlewares/ST/touchgfx/framework/source/touchgfx/widgets/canvas/CWRVectorRenderer.cpp</FilePath>
    </File>
  </Files>
</Group>
```

## 问题七：点击返回按钮卡顿 2-3 秒

**现象**：SensorPage 数据显示正常，但点击返回按钮后，界面卡住 2-3 秒才跳转到 HomePage。

**原因**：`SensorPageView::updateSensorInfo()` 中有一个未限频的 `printf` 调用，每次 Model tick 都会触发。在 STM32 上，`printf` 使用 UART 轮询模式输出（每字符阻塞等待 TX 寄存器就绪），约 60 字符/次 × 每次 tick ≈ 5ms+ 的 CPU 阻塞时间。高频率调用导致 UART 发送积压，主循环被严重拖慢，返回事件无法及时处理。

**解决**：移除所有 UI 帧路径中的调试 `printf` 调用。

**修改文件**：

1. `TouchGFX/gui/src/sensorpage_screen/SensorPageView.cpp`：
   - 移除 `updateSensorInfo()` 中的 `printf("[Sensor] T:%.1f ...")`
   - 移除 `setupScreen()` 中的 `printf("[SensorPage] setupScreen OK")`
   - 移除 `#include <stdio.h>`

   ```cpp
   void SensorPageView::updateSensorInfo(float temperature, float humidity,
                                          uint16_t co2, uint32_t heartRate, uint32_t spo2)
   {
       Unicode::snprintfFloat(valueTempBuffer, VALUETEMP_SIZE, "Tem: %.1f C", temperature);
       valueTemp.invalidate();
       // ... 其余 snprintf + invalidate（无 printf）
   }
   ```

2. `TouchGFX/target/generated/TouchGFXConfiguration.cpp`：
   - 移除 `touchgfx_taskEntry()` 中的 3 处 printf
   - 移除 `#include <stdio.h>`

   ```cpp
   void touchgfx_taskEntry()
   {
       static uint32_t entryCnt = 0;
       entryCnt++;
       if (OSWrappers::isVSyncAvailable())
       {
           hal.backPorchExited();
       }
   }
   ```

3. `Core/Src/main.c`：
   - 移除主循环中的 `printf("[LOOP] ...")` 和 `loopCnt` 变量
