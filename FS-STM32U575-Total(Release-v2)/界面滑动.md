# 界面滑动 — HomePage SwipeContainer 问题全记录

## 核心方案

按钮**不放**入 SwipeContainer 页面内（会导致黑屏），而是放在 View 层级，通过 `handleTickEvent()` 监控 `getSelectedPage()` 控制显隐。

## 问题 1：按钮放入 SwipeContainer Page1 导致黑屏

**现象**：把 `WirelessConnection`、`toggleButton4`、`toggleButton1` 加到 `swipeContainer1Page1` 内部后，屏幕黑屏（有背光无画面）。

**根因**：TouchGFX SwipeContainer 内部放置 ToggleButton 有渲染冲突。

**解决**：按钮留在 View 层级（`add()` 到 View，不是 `swipeContainer1Page1.add()`），在 `HomePageView.cpp` 的 `handleTickEvent()` 中根据当前页面控制显隐。

**改动文件**：

[`HomePageView.hpp`](gui/include/gui/homepage_screen/HomePageView.hpp) — 新增 `lastPage` 成员：
```cpp
protected:
    uint8_t textBuf[200];
    int lastPage;  // 追踪当前页面，避免每帧重复 setVisible
```

[`HomePageView.cpp`](gui/src/homepage_screen/HomePageView.cpp) — 页面变化时切换按钮显隐：
```cpp
HomePageView::HomePageView()
    : lastPage(0)
{
}

void HomePageView::handleTickEvent()
{
    int page = swipeContainer1.getSelectedPage();
    if (page != lastPage)
    {
        lastPage = page;
        bool show = (page == 0);
        WirelessConnection.setVisible(show);
        toggleButton4.setVisible(show);
        toggleButton1.setVisible(show);
        WirelessConnection.invalidate();
        toggleButton4.invalidate();
        toggleButton1.invalidate();
    }
}
```

**效果**：第 0 页显示按钮，滑动到其他页按钮自动隐藏（触摸也一并禁用）。

## 问题 2：Designer "Generate Code" 后三个必做修复

每次 TouchGFX Designer 点击 "Generate Code" 后，以下三个文件会被重置，需要手动修：

### 2a. SwipeContainer 尺寸丢失 → 黑屏

**文件**：[`HomePageViewBase.cpp`](generated/gui_generated/src/homepage_screen/HomePageViewBase.cpp)

Designer 生成的 `swipeContainer1` 没有 `setWidth(320)` / `setHeight(240)`，导致容器 0x0，黑屏。

```cpp
swipeContainer1.setXY(0, 0);
swipeContainer1.setWidth(320);   // ← Designer 会删掉这行
swipeContainer1.setHeight(240);  // ← Designer 会删掉这行
swipeContainer1.setPageIndicatorBitmaps(...);
```

### 2b. WiFiModalLink Z-order 错误

**文件**：同上

Designer 把 `add(WiFiModalLink)` 放在 `add(swipeContainer1)` **之前**，导致 modal 在 SwipeContainer 后面，弹窗不可见/不可点击。

**修法**：`add(WiFiModalLink)` 必须放在构造函数**最后**（所有按钮之后）。

### 2c. toggleButton4 跳转目标错误

**文件**：同上

Designer 把 `toggleButton4` 的跳转改为 `gotoApplicationPageScreenNoTransition()`，需要改回 `gotoSensorPageScreenNoTransition()`。

## 问题 3：SensorPage 过渡方法被 Designer 删除

**现象**：编译报错 `no member named 'gotoSensorPageScreenNoTransition' in 'FrontendApplication'`

**根因**：Designer 只为有交互配置的页面生成过渡方法。SensorPage 若无交互，就不会在 `FrontendApplicationBase` 中生成 `gotoSensorPageScreenNoTransition()`。

**解决**：将 SensorPage 过渡方法永久放在用户代码中（`gui/` 目录，Designer 不碰）。

**改动文件**：

[`FrontendApplication.hpp`](gui/include/gui/common/FrontendApplication.hpp)：
```cpp
class FrontendApplication : public FrontendApplicationBase
{
public:
    // ... 原有代码 ...

    // SensorPage transition — permanent in user code (Designer may drop from base)
    void gotoSensorPageScreenNoTransition();

private:
    touchgfx::Callback<FrontendApplication> sensorPageTransitionCallback;
    void gotoSensorPageScreenNoTransitionImpl();
};
```

[`FrontendApplication.cpp`](gui/src/common/FrontendApplication.cpp)：
```cpp
#include <gui/sensorpage_screen/SensorPageView.hpp>
#include <gui/sensorpage_screen/SensorPagePresenter.hpp>
#include <touchgfx/transitions/NoTransition.hpp>

void FrontendApplication::gotoSensorPageScreenNoTransition()
{
    sensorPageTransitionCallback = touchgfx::Callback<FrontendApplication>(
        this, &FrontendApplication::gotoSensorPageScreenNoTransitionImpl);
    pendingScreenTransitionCallback = &sensorPageTransitionCallback;
}

void FrontendApplication::gotoSensorPageScreenNoTransitionImpl()
{
    touchgfx::makeTransition<SensorPageView, SensorPagePresenter,
        touchgfx::NoTransition, Model>(
        &currentScreen, &currentPresenter, frontendHeap,
        &currentTransition, &model);
}
```

## 问题 4：SensorPage 通配符文本显示 "New Text" 而非传感器数据

**现象**：SensorPage modal 弹窗显示 "New Text" / "New Text1"，不显示真实传感器数据。

**根因**：Designer 生成的文本（`texts.xml`）是普通字符串 "New Text"，不含通配符占位符。`TextAreaWithOneWildcard` 需要 typed text 里有通配符才能显示 wildcard buffer。

**解决**：

**4a.** [`texts.xml`](assets/texts/texts.xml) — 将四行文本改为通配符占位符：
```xml
<!-- 改前 -->
<Translation Language="GB">New Text</Translation>
<Translation Language="GB">New Text1</Translation>

<!-- 改后 -->
<Text Id="__SingleUse_1AWP"> <Translation Language="GB">&lt;Heart&gt;</Translation> </Text>
<Text Id="__SingleUse_JR7L"> <Translation Language="GB">&lt;CO2&gt;</Translation> </Text>
<Text Id="__SingleUse_NDVJ"> <Translation Language="GB">&lt;Hum&gt;</Translation> </Text>
<Text Id="__SingleUse_RBVJ"> <Translation Language="GB">&lt;Temp&gt;</Translation> </Text>
```

**4b.** [`Texts.cpp`](generated/texts/src/Texts.cpp) — 将字符串原地替换为通配符字符 `0x02`（和工程中其他通配符文本一致），**保持字节数不变以维护偏移量表**：
```cpp
// 改前
0x4e, 0x65, 0x77, 0x20, 0x54, 0x65, 0x78, 0x74, 0x31, 0x0, // @72 "New Text1"
0x4e, 0x65, 0x77, 0x20, 0x54, 0x65, 0x78, 0x74, 0x0,    // @92 "New Text"

// 改后
0x2, 0x0, 0x0, 0x0, 0x0, 0x0, 0x0, 0x0, 0x0, 0x0, // @72 "<Temp>"
0x2, 0x0, 0x0, 0x0, 0x0, 0x0, 0x0, 0x0, 0x0,       // @92 wildcard
```

> **注意**：`0x02` 是 Typography "Default" 的通配符字符。偏移量 72 和 92 不变，所以 `LanguageGb.cpp` 无需修改。

## 问题 5：SensorPageViewBase 重复 `add(__background)` 导致 HardFault

**现象**：点击 toggleButton4 跳转 SensorPage 时按钮卡住不动。

**根因**：Designer bug — `SensorPageViewBase.cpp` 中 `__background` 被 add 两次。

**解决**：运行 `fix_after_designer.py`（Fix 1），自动检测并删除重复块。

---

## 自动化脚本：`fix_after_designer.py`

**位置**：[`fix_after_designer.py`](fix_after_designer.py)

**用法**：每次 Designer "Generate Code" 后运行：
```bash
cd TouchGFX
python fix_after_designer.py
```

**功能**：
| Fix | 说明 |
|-----|------|
| Fix 1 | 删除 `SensorPageViewBase.cpp` 中重复的 `add(__background)` |
| Fix 2 | 将 `FrontendApplicationBase.cpp` 中 `&FrontendApplication::` 改为 `&FrontendApplicationBase::` |

---

## Designer "Generate Code" 后完整检查清单

1. `python fix_after_designer.py`
2. `HomePageViewBase.cpp`：补 `setWidth(320)` / `setHeight(240)`，`add(WiFiModalLink)` 移到最后，`toggleButton4` 回调改回 `gotoSensorPageScreenNoTransition()`
3. `texts.xml` 四个传感器文本是否被重置为 "New Text"
4. `Texts.cpp` 字符串是否被重置
5. `FrontendApplicationBase.hpp/.cpp` 是否缺 SensorPage 过渡方法（已在用户代码中永久保留，不影响）

---

## 关键原则

> **按钮不放 SwipeContainer 页面内。用 View 层 + handleTickEvent 控制显隐。**
>
> **过渡方法放用户代码 `FrontendApplication`，不依赖 Designer 生成。**
>
> **每次 Designer "Generate Code" 后跑脚本 + 手动检查。**
