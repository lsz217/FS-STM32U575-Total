## 一、图片中文文件名导致 Generate Assets 失败

### 问题

Imageconverter 只接受 ASCII 文件名，中文命名的 PNG 文件报错。

### 解决

重命名为 ASCII 文件名：

| 原文件 | 改为 |
|--------|------|
| `jimeng-2026-05-16-6003-...` | `modal_temp_bg.png` |
| `jimeng-2026-05-16-6060-...` | `modal_hum_bg.png` |
| `jimeng-2026-05-16-7531-...` | `modal_co2_bg.png` |
| `jimeng-2026-05-16-8263-...` | `modal_heart_bg.png` |

删除不再使用的文件：`mti-温度.png`、`二氧化碳.png`、`心率.png`、`湿度.png`、`空气温度.png`

同时在 `.touchgfx` JSON 中将 ModalWindow 的 FileName 改为新名字，然后关闭再重新打开 Designer 让它加载修改，再 Generate Assets。

## 二、生成代码出现中文 Bitmap ID

### 问题

Designer 根据中文文件名生成了乱码 Bitmap ID，编译报错：

```
error: use of undeclared identifier 'BITMAP_JIMENG_2026_05_16_6060_...'
```

### 解决

手动修改 `SensorPageViewBase.cpp`，替换为正确的 ID：

```cpp
// 错误（Designer 生成）
modalWindow2.setBackground(touchgfx::BitmapId(BITMAP_JIMENG_2026_05_16_6060_...), 0, 0);
modalWindow3.setBackground(touchgfx::BitmapId(BITMAP_JIMENG_2026_05_16_7531_...), 0, 0);

// 正确（手动修改）
modalWindow2.setBackground(touchgfx::BitmapId(BITMAP_MODAL_HUM_BG_ID), 0, 0);
modalWindow3.setBackground(touchgfx::BitmapId(BITMAP_MODAL_CO2_BG_ID), 0, 0);
```

## 三、TextArea 类型不匹配

### 问题

Designer 生成 `TextArea`（无 wildcard），但 View 需要 `TextAreaWithOneWildcard` 来动态显示传感器数值。编译报错 8 个 `undeclared identifier`。

### 解决

**头文件** `SensorPageViewBase.hpp` 中：

```cpp
// 1. 添加头文件
#include <touchgfx/widgets/TextAreaWithWildcard.hpp>

// 2. 成员类型改为 TextAreaWithOneWildcard
touchgfx::TextAreaWithOneWildcard textArea1;  // 原来: touchgfx::TextArea

// 3. 添加 wildcard 缓冲区（让 View 写入传感器数值）
static const uint32_t TEXTAREA1_SIZE = 30;
touchgfx::Unicode::UnicodeChar textArea1Buffer[TEXTAREA1_SIZE];
// textArea2/3/4 同理，TEXTAREA4_SIZE = 40
```

**源文件** `SensorPageViewBase.cpp` 中，为每个 TextArea 添加 `setWildcard()`：

```cpp
textArea1.setTypedText(touchgfx::TypedText(T___SINGLEUSE_RBVJ));
textArea1.setWildcard(textArea1Buffer);   // ← 必须添加，关联缓冲区
```

**作用**：`Unicode::snprintfFloat(textArea1Buffer, ...)` 写入缓冲区后调用 `invalidate()` 即可刷新显示，无需重建整个 TextArea。

## 四、Designer 重新生成覆盖手动修改

### 问题

每次点 Designer 的 "Generate Code" 都会覆盖 `generated/gui_generated/` 下的文件，撤销所有手动修改。

### 解决

1. **不要点 "Generate Code"**，除非需要重新生成画面布局
2. 如果必须生成，生成后重新修改这两个文件：
   - `generated/gui_generated/include/.../SensorPageViewBase.hpp`
   - `generated/gui_generated/src/.../SensorPageViewBase.cpp`
3. 生成的文件是只读的，修改前先解除：

```bash
chmod +w SensorPageViewBase.hpp SensorPageViewBase.cpp
```

## 五、TextArea 应放在 ModalWindow 内部

### 问题

最初 TextArea 直接 `add(textArea1)` 到 View 上，需要手动管理显示/隐藏，且坐标是屏幕绝对坐标。

### 解决

改为 `modalWindow.add(textArea)` 模式（参照 startuppage）：

```cpp
// 正确：textArea 作为 modalWindow 的子控件
textArea1.setXY(105, 136);
textArea1.setWildcard(textArea1Buffer);
modalWindow1.add(textArea1);  // 添加到 modal 内部
add(modalWindow1);            // modal 添加到 View

// 错误（已废弃）：
// add(textArea1);  // 直接添加到 View，需手动控制 visible
```

**作用**：ModalWindow 隐藏时内部控件自动隐藏，无需手动 `setVisible(false)`，简化 View 代码。

## 六、show/hide 改用 setVisible 模式

### 问题

最初使用 `modalWindow.show()/hide()` + 每次遍历 4 个 Modal，效率低。

### 解决

改为 `setVisible(true/false)` + `hideActiveModal()` 精准控制（模仿 startuppage 风格）：

```cpp
// SensorPageView.hpp 新增成员
private:
    void hideActiveModal();      // 只隐藏当前活跃的 modal
    int8_t activeModal;          // -1=无, 0=温度, 1=湿度, 2=CO2, 3=心率

// SensorPageView.cpp — 精准隐藏
void SensorPageView::hideActiveModal()
{
    if (activeModal == 0)
        modalWindow1.setVisible(false);
    else if (activeModal == 1)
        modalWindow2.setVisible(false);
    else if (activeModal == 2)
        modalWindow3.setVisible(false);
    else if (activeModal == 3)
        modalWindow4.setVisible(false);
}

// Toggle 模式：点同一个图标→关闭，点不同图标→切换
void SensorPageView::showTemp()
{
    if (activeModal == 0)
    {
        modalWindow1.setVisible(false);
        activeModal = -1;
    }
    else
    {
        if (activeModal >= 0)
            hideActiveModal();      // 只关当前活跃的，不遍历全部
        modalWindow1.setVisible(true);
        activeModal = 0;
    }
    invalidate();
}
```

**作用**：
- `hideActiveModal()` 只操作 1 个 Modal，而非每次遍历 4 个
- Toggle 逻辑：点击已打开的图标关闭，点击不同图标先关旧再开新
- `setVisible(true/false)` 与 startuppage 风格一致

## 七、SVG 图标改为图片

### 问题

SVG 渲染依赖 `CanvasWidgetRenderer`，占用缓冲区资源，且 FlexButton 覆盖 SVG 时点击无响应。

### 解决

将 4 个 SVG 图标替换为普通位图（ScalableImage），文件从 `images/__generated` 导入 PNG：

```cpp
// 原来：SVGImage（需要 CanvasWidgetRenderer::setupBuffer）
svgImage1.setSVG(SVG_TEMP_ICON_ID);

// 改为：ScalableImage（普通位图，无需额外缓冲区）
scalableImage1.setBitmap(touchgfx::Bitmap(BITMAP_HUMIDITY_1_ID));
scalableImage1.setPosition(0, 0, 55, 55);
scalableImage1.setScalingAlgorithm(touchgfx::ScalableImage::NEAREST_NEIGHBOR);
```

**作用**：
- 移除对 `CanvasWidgetRenderer` 的依赖（不再需要 `setupBuffer(resetBuffer` 和 4800 字节 canvasBuffer）
- 图标作为普通位图渲染，性能更好，不干扰触摸分发

## 八、FlexButton 改为最小配置

### 问题

原来的 FlexButton 配置了 `setBoxWithBorderColors/setBorderSize/setAlpha(0)`，子控件 `borderBox` 可能干扰触摸。

### 解决

移除所有视觉属性，只保留必要配置：

```cpp
flexButton1.setBoxWithBorderPosition(0, 0, 0, 0);  // 宽高为 0，不显示边框
flexButton1.setAlpha(255);       // 完全不透明（但边框尺寸为 0 所以不可见）
flexButton1.setPosition(5, 0, 50, 50);  // 只靠位置和大小接收触摸
flexButton1.setAction(flexButtonCallback);
```

**作用**：FlexButton 成为纯触摸检测区域，无任何可见元素。

## 九、回调分发逻辑

### 作用

`flexButtonCallbackHandler` 是所有 4 个 FlexButton 共享的回调入口，通过比较地址 `&src == &flexButtonN` 分发到对应的虚函数。虚函数由派生类 `SensorPageView` 重写，实现具体的 Modal 切换逻辑。

```
flexButton1(湿度图标) → showHum()
flexButton2(CO2图标)  → showCO2()
flexButton3(温度图标) → showTemp()
flexButton4(心率图标) → showHeartRate()
```

---

## 十、FlexButton 与 Button 回调类型对比

| 控件 | 回调签名 |
|------|---------|
| Button | `Callback<T, const AbstractButton&>` |
| FlexButton (BoxWithBorderButtonStyle\<ClickButtonTrigger\>) | `Callback<T, const AbstractButtonContainer&>` |
| TouchArea | `Callback<T, const AbstractButton&>` |

两者不兼容，FlexButton 的 handler 签名必须是 `void handler(const AbstractButtonContainer& src)`。
