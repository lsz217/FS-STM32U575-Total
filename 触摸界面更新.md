## 问题 1：屏幕完全无显示（连 `--` 都没有）

### 原因

三个叠加问题：

**1.1 ModalWindow 用了 `setVisible()` 而非 `show()`/`hide()`**

`ModalWindow` 需要 `show()`/`hide()` 才能正确管理遮罩层（shade）和输入捕获。原代码使用 `setVisible(true)`/`setVisible(false)` 只设置了 visibility flag，未触发 Modal 的完整显示逻辑。

**1.2 TypedText 模板导致 textArea 宽度为 0**

TypedText 模板是 `"<>"`（裸通配符，无前后文字），TouchGFX 自动计算的 textArea 宽度接近零，所有文字被裁剪。

**1.3 Unicode 格式化字符串问题**

- `Unicode::snprintfFloat` 不支持 `%%` 转义，湿度格式 `"Hum: %.1f %%"` 会格式化失败
- `Unicode::snprintf`/`snprintfFloat` 返回 `UnicodeChar*` 而非 `int`，不能直接赋值给 `int len`
- CO2/心率用了 `%d`（有符号），应改为 `%u`（无符号）

### 解决

```cpp
// 1. 所有 show*/hideActiveModal 改用 show()/hide()
modalWindow1.show();   // 替代 setVisible(true)
modalWindow1.hide();   // 替代 setVisible(false)

// 2. setupScreen() 中显式设置 textArea 宽高
textArea1.setPosition(105, 136, 200, 30);  // Tem
textArea2.setPosition(93, 109, 200, 30);   // Hum
textArea3.setPosition(15, 175, 200, 30);   // CO2
textArea4.setPosition(5, 186, 310, 30);    // HR/SpO2

// 3. 湿度/心率格式化：先 snprintf 数值，再手动追加 " %"
Unicode::snprintfFloat(textArea2Buffer, TEXTAREA2_SIZE, "Hum: %.1f", humidity);
uint32_t len = Unicode::strlen(textArea2Buffer);
textArea2Buffer[len] = ' '; textArea2Buffer[len+1] = '%'; textArea2Buffer[len+2] = '\0';

// 4. 无符号类型用 %u
Unicode::snprintf(textArea3Buffer, TEXTAREA3_SIZE, "CO2: %u ppm", co2);
```

## 问题 2：手指未放上 MAX30102 时心率/血氧显示噪声数据

### 原因

`Model.cpp:119` 传心率/血氧时未判断 `ch_hr_valid` / `ch_spo2_valid` 有效性标志，直接把 MAX30102 的原始值（无手指时是随机噪声）传给了 View。

对比 `updateHeartRateInfo` 那条路径（122-129行）有 `if(ch_hr_valid || ch_spo2_valid)` 判断，但 SensorPage 的转发路径没有。

### 解决

```cpp
// Model.cpp:119 → 仅在有效时才传实值，否则传 0xFFFFFFFF 作为 sentinel
modelListener->updateSensorInfo(gTemRH_Val.Tem, gTemRH_Val.Hum, gSCD41_Val.CO2,
    ch_hr_valid   ? (uint32_t)(n_heart_rate / 4) : 0xFFFFFFFF,
    ch_spo2_valid ? (uint32_t)n_sp02             : 0xFFFFFFFF);
```

View 端收到 `0xFFFFFFFF` 时心率/血氧显示 `--`，温度/湿度/CO2 不受影响（环境传感器始终有值）。

## 问题 3：手指移开后立即显示 `--`

### 原因

上一版的修改让 `updateSensorInfo` 收到 sentinel 后立即显示 `--`，没有保持机制。

### 需求

- 手指放上 → 显示实时值
- 手指移开 → 保持最后有效值显示 15 秒
- 15 秒内手指放回 → 恢复实时显示
- 15 秒后仍无有效数据 → 回到 `--`

### 解决

利用 `updateSensorInfo` 每帧都会被 Model::tick() 调用一次的特点，用帧计数器代替定时器：

```cpp
// SensorPageView.hpp 新增成员
int32_t hrInvalidCount; // 0=收到有效数据, >0=连续无效帧数

// updateSensorInfo() 心率/血氧逻辑
if (hrValid || spo2Valid)
{
    hrInvalidCount = 0;          // 有效数据：计数器归零，显示实时值
    // ... 正常显示逻辑 ...
}
else
{
    hrInvalidCount++;            // 无效数据：累加帧数
    if (hrInvalidCount == 900)   // 900帧 ≈ 15秒 @60fps
    {
        // 超时：切回 "--"
        Unicode::strncpy(textArea4Buffer, "HR: -- bpm  SpO2: --", TEXTAREA4_SIZE);
    }
    // 未超时：保持显示最后有效值（不更新 buffer）
}
```

## 完整状态流转

```
         ┌──────────┐
         │  显示 --  │ ◄──────────────┐
         └────┬─────┘                │
              │ 手指放上，收到有效值    │ 900帧无有效数据
              ▼                      │
         ┌──────────┐   手指移开     ┌─┴────────┐
         │ 显示实值  │ ────────────→ │ 倒计时中   │
         │ counter=0 │              │ counter++ │
         └──────────┘ ◄──────────── └──────────┘
              ▲         手指放回（counter=0）
              
温度、湿度、CO2 不受此机制影响，始终实时更新。
```

## 问题 4：HomePage 扩按钮跳转到 SensorPage

### 需求

HomePage 有 `toggleButton4`（扩按钮），原本跳转 ApplicationPage，需要改为跳转 SensorPage。

### 涉及文件

| 文件 | 修改内容 |
|------|----------|
| `TouchGFX/gui/include/gui/homepage_screen/HomePageView.hpp` | 添加 `expandButtonCallback` 和 `expandButtonCallbackHandler` |
| `TouchGFX/gui/src/homepage_screen/HomePageView.cpp` | `setupScreen()` 中重绑定 toggleButton4 的 action，实现跳转 SensorPage |
| `TouchGFX/FS-STM32U575-Total.touchgfx` | Interaction4 目标从 `ApplicationPage` 改为 `SensorPage` |

### 实现方式

不能直接修改生成的 `HomePageViewBase.cpp`（只读 + DO NOT MODIFY），改为在用户层覆盖：

```cpp
// HomePageView.hpp 新增私有成员
touchgfx::Callback<HomePageView, const touchgfx::AbstractButton&> expandButtonCallback;
void expandButtonCallbackHandler(const touchgfx::AbstractButton& src);

// HomePageView.cpp — setupScreen() 中覆盖基类的 action
void HomePageView::setupScreen()
{
    HomePageViewBase::setupScreen();
    // toggleButton4（扩按钮）→ SensorPage，覆盖基类的 ApplicationPage 路由
    toggleButton4.setAction(expandButtonCallback);
}

void HomePageView::expandButtonCallbackHandler(const touchgfx::AbstractButton& src)
{
    application().gotoSensorPageScreenNoTransition();
}
```

### 原理

`setAction()` 可以重复调用，后绑定的回调覆盖前一个。基类构造函数先绑定 `buttonCallback`（路由到 ApplicationPage），`setupScreen()` 中重新绑定到 `expandButtonCallback`（路由到 SensorPage），最终执行用户回调。

## 问题 5：Designer 重新生成导致全屏黑 + SensorPage 布局崩溃

### 原因

`.touchgfx` 文件被 Designer 修改后重新生成代码，`SensorPageViewBase.cpp` 出现三个严重问题：

**5.1 `__background` 被重复 `add()` — 导致程序启动 crash → 黑屏**

```cpp
// 生成的错误代码
__background.setPosition(0, 0, 320, 240);
__background.setColor(touchgfx::Color::getColorFromRGB(0, 0, 0));
add(__background);

__background.setPosition(0, 0, 320, 240);  // 重复块！
__background.setColor(touchgfx::Color::getColorFromRGB(0, 0, 0));
add(__background);  // 重复 add → widget 链表损坏 → crash
```

同一个 widget 重复 `add()` 导致 TouchGFX 内部显示链表损坏，应用启动即 crash，屏幕全黑无反应。

**5.2 `textArea1-4` 被移到 ModalWindow 外部**

```cpp
// 正确的原始结构
modalWindow1.add(textArea1);
add(modalWindow1);

// Designer 生成错误结构
add(textArea1);       // textArea 加到主视图
add(modalWindow1);    // modal 也加到主视图，两者没有父子关系
```

textArea 不在 modal 内，导致弹窗时底层图标透过来，且 modal 内没有文本显示。

**5.3 `flexButton1-4` 丢失 `setAction()` 回调**

```cpp
// 正确
flexButton1.setAction(flexButtonCallback);

// Designer 生成
flexButton1.setPosition(5, 0, 50, 50);
add(flexButton1);
// ← 没有 setAction！点击无反应
```

### 解决

从 git 恢复 `SensorPageViewBase.hpp` / `SensorPageViewBase.cpp` 到 Designer 破坏前的版本，恢复 `SensorPageView.hpp` / `SensorPageView.cpp` 到用户改进版本（保留 close 按钮、sentinel 处理、hold-over 逻辑）。

**核心教训：Designer 重新生成代码后必须逐文件检查 diff，不能假设生成的代码总是正确的。**

## 问题 6：textArea 只显示模板文字不显示传感器数据

### 原因

TypedText 数据库被 Designer 覆写：

```
T___SINGLEUSE_RBVJ: "New Text1"    ← 纯文本，无 %s 通配符
T___SINGLEUSE_NDVJ: "New Text"     ← 同上
T___SINGLEUSE_JR7L: "New Text"     ← 同上
T___SINGLEUSE_1AWP: "New Text"     ← 同上
```

`TextAreaWithOneWildcard` 需要 TypedText 模板包含 `%s`（或 `<>`）作为通配符占位，buffer 内容通过替换 `%s` 来显示。模板是纯文本 `"New Text"` 时不包含通配符，buffer 的传感器数据无法注入，textArea 永远显示静态的 "New Text" 而不是 "Tem: 25.3 C"。

### 解决方向

需要让文字模板包含 `%s` 通配符。在 TouchGFX Designer 中将对应 TextArea 的 Wildcard1 配置为 `Use Buffer = true` 并设置合适的 BufferSize，Designer 生成的 TypedText 会自动使用 `<>` 模板。或者在文字数据库中手动将这四个 key 的值改为 `"<>"`。

### 正确的 TextAreaWithOneWildcard 完整流程

```cpp
// 1. TypedText 模板包含 %s（DB 中是 "<>"）
// 2. 基类中绑定模板和缓冲区
textArea1.setTypedText(touchgfx::TypedText(T___SINGLEUSE_RBVJ));  // 模板："<>"
textArea1.setWildcard(textArea1Buffer);                            // 缓冲区指针
// 3. 用户代码中写入缓冲区
Unicode::snprintfFloat(textArea1Buffer, TEXTAREA1_SIZE, "Tem: %.1f C", temperature);
textArea1.invalidate();  // 触发重绘
```
